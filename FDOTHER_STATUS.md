## RLE解码算法要点

基于反编译函数 `sub_4E98D` 实现，关键点：

1. **优先级修复**: `(((4 * value) & 0xFF) >> 2) + 1` 而不是 `>> (2 + 1)`

2. **四种模式** (每行独立处理):
   - bit7=1, bit6=1: 跳转模式 (dst前移 count_1 像素)
   - bit7=1, bit6=0: 复制模式 (从src复制 count_1 字节)
   - bit7=0, bit6=0: 填充模式 (用下一字节填充 count_1 像素)
   - bit7=0, bit6=1: 跳转 (继续外层循环)

3. **count_1 计算**: `(4 * value >> 2) + 1` = `value + 1`

## LMI1类型新发现

### 结构
- 字节0-3: `LMI1` 头
- 字节4-5: offset_count (uint16)
- 字节6+: offset表 (每entry 2字节) → 指向子图形位置表
- offset表后的数据区: uint32指针表，指向各个子图形

### 关键代码
`sub_16886` 中的公式: `*(_DWORD *)(a7 + 4 * a8 + 6) + a7`
- a7 = 资源起始地址
- a8 = 索引号
- 结果 = 子图形指针表位置
