# ANI.DAT AFM 格式分析文档

## 概述

ANI.DAT 是 FD2 游戏的动画数据文件，包含多个 AFM (Animation File Manager) 资源。每个 AFM 资源是一个独立的动画片段。

## 文件结构

### 文件头
- 魔数: 6 字节 "LLLLLL"
- 索引表: 从偏移 6 开始，每条目 4 字节（32位偏移）

### AFM 资源索引
```
偏移 6:  AFM 0 偏移 (4 字节)
偏移 10: AFM 1 偏移 (4 字节)
偏移 14: AFM 2 偏移 (4 字节)
...
```

## AFM 资源结构

### AFM 头部 (173 字节)
- 偏移 165-166: 帧数 (WORD, 小端)
- 偏移 0-164: 动画元数据/描述
- 偏移 167-172: 保留

### 帧结构
每帧由 8 字节头部 + 帧数据组成：
```
偏移 0-1: size (WORD) - 帧数据大小
偏移 2-3: param (WORD) - 命令数量
偏移 4-7: 保留
偏移 8+: 帧数据 (命令流)
```

## 命令格式

帧数据是命令流，由 `param` 个命令组成。每个命令以命令字节开头。

### 命令 0x00 - 填充调色板
```
[00][color]
```
用单一颜色填充整个调色板（768字节）。

### 命令 0x01 - 复制调色板
```
[01][palette_data: 768 bytes]
```
直接复制 768 字节调色板数据。

### 命令 0x02 - RLE 解码调色板
```
[02][rle_data...]
```
RLE 解码调色板数据。

### 命令 0x04 - 填充像素缓冲区
```
[04][color]
```
用单一颜色填充整个像素缓冲区（64000字节）。

### 命令 0x05 - 复制像素数据
```
[05][pixel_data: 64000 bytes]
```
直接复制 64000 字节像素数据。

### 命令 0x06 - RLE 解码像素
```
[06][rle_data...]
```
RLE 解码像素数据。

RLE 格式：
- 如果 `(byte & 0xC0) == 0xC0`: 计数模式
  - count = byte & 0x3F
  - 下一个字节是颜色值
  - 写入 count 个颜色值
- 否则: 直接写入该字节作为颜色值

### 命令 0x07 - 点绘制
```
[07][count:WORD][offset:WORD][color:BYTE]...
```
绘制 count 个点，每个点由 offset 和 color 组成。

### 命令 0x08 - 填充段
```
[08][count:WORD][offset:WORD][size:BYTE][color:BYTE]...
```
绘制 count 个段，每段从 offset 开始，填充 size 个 color。

### 命令 0x09 - 复制数据
```
[09][count:WORD][dst:WORD][size:BYTE][data: size bytes]...
```
复制 count 个数据块，每个数据块从帧数据复制到像素缓冲区的 dst 位置。

## 缓冲区

### 调色板缓冲区
- 大小: 768 字节 (256 × 3)
- 格式: RGB 三字节一组，DOS 6-bit (0-63)

### 像素缓冲区
- 大小: 64000 字节 (320 × 200)
- 格式: 索引像素，指向调色板

## 调色板转换

DOS 调色板是 6-bit (0-63)，需要转换到 8-bit (0-255)：
```
RGB_8bit = min(255, RGB_6bit × 4)
```

## 解码流程

1. 读取 AFM 索引，获取所有 AFM 偏移
2. 对每个 AFM：
   - 读取帧数（偏移 165-166）
   - 初始化调色板和像素缓冲区
   - 逐帧处理命令流
   - 生成图像帧
3. 输出 GIF 动画
